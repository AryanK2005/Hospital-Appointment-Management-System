Hospital Appointment Management System
#Overview

The Hospital Appointment Management System is a Java console-based mini project designed to simulate real-world hospital appointment handling. It allows hospitals or clinics to register patients and doctors, book appointments, prioritize emergency cases, and store appointment records securely. The project demonstrates practical usage of core Java concepts taught in academic courses.

# Problem Statement

In many small hospitals and clinics, appointment management is often handled manually, which can lead to scheduling conflicts, lack of prioritization for emergency patients, and poor record maintenance. This project addresses these issues by providing a simple, structured, and automated appointment management solution.

# Features
Register patients and doctors
Book appointments with emergency priority handling
View appointments in priority order
Background reminder using multithreading
Save appointment data to a file
Robust error handling with custom exceptions

# Java Concepts Used
Object-Oriented Programming (Encapsulation, Classes, Objects)
Collections Framework (ArrayList, PriorityQueue)
Exception Handling (custom checked exceptions)
Multithreading (Thread)
File Handling (FileWriter)
Console-based user interaction

# How to Run the Project
Clone the repository or download the source code.
Open the project in any Java-supported IDE (VS Code, IntelliJ, Eclipse).

Compile and run the file:
javac HospitalApp.java
java HospitalApp
Follow the on-screen menu options.

# Output
Appointments are saved in a file named appointments.txt
Emergency appointments are automatically prioritized
